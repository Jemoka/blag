Notably, the entire file system only supports \(2^{16} = 32MB\) worth of space due to short address types.

For each file on the disk, we store payload data in a bunch of places scattered across the disk, and a single inode which stores the location of each block for the file in an array.

  • inodes contain an ordered list of block numbers, file size, permissions.
  • all inodes are stored together in an inode table, which starts at block 2. Blocks 0 and 1 are disk metadata.
  • inode can be read into memory individally to cache
  • 10% of harddrive is used to inode

Unix V6 Filesystem limits the maximum file size in order to keep the inode a finite size.

The inode table for each file only contains space to point to \(8\) block. 1 block = 1 sector on Unix v6. inodes are usualy 32 bytes big, and 1 block = 1 sector = 512 bytes. usually this packs 16 inodes per block

inodes are 1 indexed in order to make.

inode

struct inode {
    uint16_t i_addr[8];
    uint16_t i_mode[8];
    uint16_t file_size;
}

Each inode contains 8 addresses in shorts in file order.

reading inode tables from disk

We read the raw 16-block inode data from the right sector, type coerce it into the inode type, and then read from it.

const size_t INODE_PER_BLOCK = SECTOR_SIZE / sizeof(struct inode);
struct inode inodes[INODE_PER_BLOCK];

char buf[SECTOR_SIZE];
readsector(2, &inodes); // recall this is the first 16 inodes: sec0 is fs info, sec1 is supernode

printf("addr: %d\n", inodes[0].i_add);

inode modes

inodes have two modes

if ((inode.i_mode & ILARG) != 0) == // node is in "large mode"
  • in small mode, the inode stores in i_addr the block numbers to the data
  • in large mode, the inode stores in the first seven numbers in i_addr block numbers to blocks that contain block numbers (512/2 = 256 block numbers, which are chars); the eighth number points to doubly indirect blocks that contain block numbers that point to other blocks

this is called indirect addressing

indirect addressing uses more steps to get to the data, and requires more blocks to get to the block numbers.

in large mode, this system can store \((7+256) \cdot (256 \cdot 512) = 34MB\), which is as large as the file system itself, which is fine now.
