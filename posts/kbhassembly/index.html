CPU review

1978 created Intel 8086.

  • Idea => C code: your job
  • C code => Assembly => Machine code: complier + assembly’s job

What we really want to learn is the skill of translating the assembly BACK into C code. We want to know how our C code is converted into machine instructions.

Breakdown

  • assembly instructions in X86 are variable length; meaning there’s each assembly command

each instruction looks like:

add $0x1, $eax
  • add is the opcode
  • %[value], like $0x1, which, because of the dollar sign, is a constant
  • %[name], like %eax, which, because of the percent sign, is a CPU scratch space

register

A register is a 64 bit scratch space that’s very fast memory in CPU; there are 16 registers. In total, therefore, there are \(64 \times 16\) bits of memory.

Memory can be loaded into a register.

Most operations require your values into registers. Most of your work involves moving memory from flash, move it into the registers, doing operations, and move it out.

One of the 16 registers is used for the program counter (PC), which is to store the address of currently executing instruction.

instruction-set architecture

ISA is a contract between the compiler and hardware, which:

  1. defines operations that the CPU can do
  2. data read/write/transfer operations
  3. controls

Intel designed the X86-64 ISA:

  • legacy support support is very bad
  • originally a 16 bit processor; then 32 bit; now 64 bit
